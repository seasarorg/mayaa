# Mayaa 自動エスケープ機能 利用者ガイド

このドキュメントは、Mayaaの自動エスケープ機能（Issue #110 由来）について、アプリケーション開発者が利用時に知っておくべき外部仕様・使い方・注意点をまとめたものです。

## 1. 概要

Mayaa テンプレートの `${}` 出力に対し、XSS脆弱性を防ぐための自動エスケープ機能を提供します。埋め込み箇所（HTML本体、属性値、JavaScript内など）ごとに適切なエスケープ方式を自動適用します。

## 2. 有効化・無効化

- グローバル設定: `service-provider.dicon` の `autoEscapeEnabled` パラメータでON/OFF
- ページ単位: HTML先頭コメント `<!-- m:autoEscape="true|false" -->` で上書き可

## 3. エスケープ動作の文脈

- HTML本体: `<`, `>`, `&` をエスケープ
- HTML属性値: 上記 + `"`, `'` をエスケープ
- SCRIPT内: JS文字列リテラルとしてエスケープ
- STYLE内: CSS値としてエスケープ
- TEXTAREA/PRE: HTML本体と同様

## 4. 非エスケープ出力

- `${=...}` または `MAYAA_SCOPE_RAW(...)` で明示的にスキップ

## 5. Scopeマクロ（script専用）

- `MAYAA_SCOPE(expr)` → JS式として埋め込み
- `MAYAA_SCOPE_AS_STRING(expr)` → JS文字列リテラル化
- `MAYAA_SCOPE_WITH_STRINGIFY(expr)` → JSONテキストとして埋め込み
- `MAYAA_SCOPE_RAW(expr)` → 非エスケープ出力

## 6. 既エスケープ値の自動検出

- 代表的なHTMLエンティティやバックスラッシュエスケープが含まれる場合は多重エスケープを抑制
- 誤検知/検出漏れが許容できない場合は `${=...}` で明示

## 7. 仕様変更点（従来との差分）

- シングルクオート `'` のエスケープ仕様を文脈ごとに明確化
  - HTML本体・TEXTAREA/PREでは `'` はエスケープしない
  - HTML属性値では `'` を `&#39;` にエスケープ
- 既存テストの期待値もこの仕様に合わせて変更

## 8. 注意事項・互換性

- デフォルトはOFF（autoEscapeEnabled=false）
- 有効化時は文脈依存で出力が変わるため、HTML文字列の厳密比較テストは差分が出る可能性あり
- 手動エスケープ済み値やJSON埋め込みは、必要に応じて `${=...}` / `MAYAA_SCOPE_RAW(...)` で明示制御
